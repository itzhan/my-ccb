# 身份模拟 —— 请求改写字段说明

> 目的:多人共号时,把每个人的真实身份痕迹「归一化」成账号绑定的一套**固定虚拟身份**,让同一个号每次请求看起来都是「同一个虚拟的人」在用。

## 三层结构

| 层 | 文件 | 职责 |
|---|---|---|
| 生成 | `src/model/identity.rs` | 造身份数据池(device_id、虚拟用户名/git名、env 预设、虚拟项目名) |
| 绑定 | `src/service/account.rs` | 账号创建时把一套身份**固化**到账号;并从真实客户端吸取版本坐标 |
| 注入 | `src/service/rewriter.rs` | 把身份真正写进发往上游的请求体 |
| 开关 | `src/service/gateway.rs` | 决定是否做归一化(账号 `identity_mode` 优先,回退全局默认) |

另有**网络层**:`src/tlsfp/tlsfp.rs` 负责 TLS ClientHello 指纹伪装(模拟 Node.js),属传输层。

---

## 一、主推理请求 `/v1/messages`

入口:`normalize_cc_identity()`(`rewriter.rs:531`)。**只动 3 处**。

### 1. `metadata.user_id`(`rewriter.rs:342`)

CC 客户端原始传入是一个 JSON 字符串:

```json
"metadata": {
  "user_id": "{\"device_id\":\"abc...\",\"account_uuid\":\"真实UUID\",\"email\":\"你的邮箱\",\"session_id\":\"...\"}"
}
```

改写后:

- `device_id` → 替换成**本账号固定的** device_id
- `account_uuid` → 统一成本账号的(或留空)
- **删掉** `email` / `org` 等多余字段(`retain` 只保留 `device_id` / `account_uuid` / `session_id`)

### 2. `system` 块文本(`rewriter.rs:620`)

用正则 `scrub()` 全量替换系统提示词里的身份痕迹:

| 原始内容 | 替换为 |
|---|---|
| `/Users/真实用户名/...` | `/Users/虚拟用户名/...` |
| `Git user: 真实姓名` | `Git user: 虚拟姓名` |
| `Platform: xxx` | `Platform: 账号绑定平台` |
| `Shell: xxx` | `Shell: 账号绑定 shell` |
| `OS Version: xxx` | `OS Version: 账号绑定 OS` |
| Windows 路径 `C:\Users\bob\...` | 归一成 `/Users/虚拟用户名/...` |
| 工作目录 / CLAUDE.md 路径 | 兜底到会话虚拟项目名 |

### 3. `messages[].content` 文本(`rewriter.rs:639`)

同样跑 `scrub()` —— 因为 `<system-reminder>` 里的 cwd、git status、CLAUDE.md 路径也会暴露身份。

> **不动的**:对话内容本身、项目子路径结构(真人本就会在多个项目里干活)。

---

## 二、遥测请求(埋点上报,非推理)

走 `rewrite_event_batch()`(`:897`)和 `rewrite_growthbook_eval()`(`:974`),改写字段更多:

- 身份:`device_id`、`email`、`account_uuid`、`organization_uuid`
- 环境:`env`(整块替换成 canonical)、`process`(内存/堆数据)
- 元数据:`additional_metadata`、`user_attributes`(GrowthBook 实验 JSON)
- **删掉**:`baseUrl` / `base_url` / `gateway`(否则暴露在用网关)

---

## 一句话总结

**真正发给模型的推理请求,核心只改一个字段 `metadata.user_id`(换 device_id/account_uuid、删 email);外加把 `system` 和 `messages` 文本里的用户名/git名/路径/平台信息正则替换成账号绑定的虚拟身份。** 遥测请求则额外清洗一批埋点字段并抹掉网关痕迹。
